Unicode import

Unicode is the most basic way fonts are applied on the web, with the following characteristics:

  • Supports dynamic adjustment of icon size, color, etc., as if adjusting fonts.
  • Does not support multicolor by default; directly adding multicolor icons will automatically remove colors.

Note: The new version of Iconfont supports two ways to use multicolor icons: SVG symbol referencing and colored font icon mode. (To use colored font icons, you need to enable the "Color" option in "Edit Project" and regenerate the project.)

Steps to use Unicode are as follows:

Step 1: Copy the generated @font-face from the project

@font-face {
  font-family: 'iconfont';
  src: url('iconfont.woff2?t=1621231825060') format('woff2'),
       url('iconfont.woff?t=1621231825060') format('woff'),
       url('iconfont.ttf?t=1621231825060') format('truetype');
}

Step 2: Define the style using iconfont

.iconfont {
  font-family: "iconfont" !important;
  font-size: 16px;
  font-style: normal;
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
}

Step 3: Select the corresponding icon and obtain the font encoding, and apply it to the page

<span class="iconfont">&#x33;</span>

"iconfont" It is the font-family of your project. You can check it by editing the project. The default is "iconfont".

font-class Reference

font-class is a variant of the Unicode usage method, designed to address the issues of Unicode being less intuitive and less semantically clear.

Compared to the Unicode method, it has the following characteristics:

  • More semantically clear and intuitive to write. It's easy to identify what the icon represents.
  • Since it uses a class to define icons, replacing an icon only requires modifying the Unicode reference inside the class.

Steps to use font-class are as follows:

Step 1: Import the generated fontclass code from your project:

<link rel="stylesheet" href="./iconfont.css">

Step 2: Select the corresponding icon, get the class name, and apply it to the page:

<span class="iconfont icon-xxx"></span>

"iconfont" is the font-family in your project. You can check this by editing the project, and the default is "iconfont".

Symbol Reference

This is a completely new usage method, and it is considered the future mainstream approach. It is also the usage currently recommended by the platform. For more details, you can refer to this article. This method essentially creates a collection of SVGs and has the following characteristics compared to the other two methods:

  • Supports multicolor icons, no longer limited to single color.
  • With some techniques, it supports styling like fonts using font-size and color.
  • Poor compatibility: supports IE9+ and modern browsers.
  • Browser rendering performance for SVG is average, even worse than PNG.

Steps to use Symbol are as follows:

Step 1: Import the generated symbol code from your project:

<script src="./iconfont.js"></script>

Step 2: Add the universal CSS code (only needs to be included once):

<style>
.icon {
  width: 1em;
  height: 1em;
  vertical-align: -0.15em;
  fill: currentColor;
  overflow: hidden;
}
</style>

Step 3: Select the corresponding icon and get the class name, and apply it to the page:

<svg class="icon" aria-hidden="true">
  <use xlink:href="#icon-xxx"></use>
</svg>
